# 💳 ربط الدفع Telr — دليل خطوة بخطوة

مرحبا صوفيان 👋 هاد الدليل غادي يمشي بيك خطوة بخطوة. خود وقتك، وكل خطوة بسيطة.

الفكرة فجملة: زدنا "موظف خفي" (ملف فمجلد `api/`) كيتواصل مع Telr بطريقة آمنة.
السر ديالك (Authentication Key) **ماغاديش يبان أبداً** فالموقع — غادي تحطّو فـ Vercel بطريقة مخفية.

---

## 📁 الملفات الجداد اللي زدنا

```
api/telr-create.js        ← "الموظف الخفي" اللي كيتواصل مع Telr
payment-success.html      ← صفحة "تم الدفع بنجاح"
payment-failed.html       ← صفحة "فشل الدفع"
vercel.json               ← إعدادات Vercel
checkout.html + checkout.js + checkout.css ← تعدّلو (زدنا خيار الدفع بالكارط)
```

---

## ✅ الخطوة 1 — رفع الملفات لـ GitHub

بما أن GitHub مربوط بـ Vercel عندك، غير زيد الملفات الجداد للـ repository ديالك:
- المجلد `api/` بكامله (فيه `telr-create.js`)
- `payment-success.html`
- `payment-failed.html`
- `vercel.json`
- بدّل: `checkout.html` · `checkout.js` · `checkout.css` (النسخ الجداد)

ملي تعمل push، Vercel غادي يحدّث الموقع أوتوماتيكياً.

---

## 🔑 الخطوة 2 — حط السر فـ Vercel (الأهم)

1. دخل لـ **vercel.com** → اختار المشروع ديال MyStamp
2. **Settings** → **Environment Variables**
3. زيد هاد 4 متغيرات (دوس "Add" لكل واحد):

| الاسم (Key) | القيمة (Value) |
|-------------|----------------|
| `TELR_STORE_ID` | `32758` |
| `TELR_AUTH_KEY` | (الـ Authentication Key ديالك من Telr) 🔑 |
| `TELR_TEST_MODE` | `1` (1 = تجريب، 0 = حقيقي) |
| `SITE_URL` | `https://mystamp.ae` |

> ⚠️ الـ `TELR_AUTH_KEY` هو السر — هنا فقط بلاصتو الآمنة. ماتحطّوش فأي ملف.

4. دوس **Save**.
5. مهم: باش المتغيرات يخدمو، خاصك تعمل **Redeploy**:
   **Deployments** → آخر deployment → الزر `···` → **Redeploy**.

---

## 🌍 الخطوة 3 — إعداد Telr (مرة وحدة)

فحساب Telr → Integrations → Hosted Payment Page → **Configuration**:

1. **"Allow tests from any IP"** → بدّلو لـ **Yes** (حيت Vercel ماعندوش IP ثابت)
2. دوس **Update**

> هادي ضرورية فالتجريب. للـ Live، إلا بغيتي أمان أكثر بالـ IP، عيّط ليا ونوريك حل.

---

## 🧪 الخطوة 4 — جرّب (بكارط تجريبية)

1. دخل لموقعك → صمّم ختم → checkout
2. عبّي المعلومات → اختار **"Pay by card"** → دوس **Pay securely**
3. غادي يوجّهك لصفحة Telr الآمنة
4. استعمل كارط التجريب ديال Telr:
   - رقم: `4000 0000 0000 0002`
   - أي تاريخ مستقبلي + أي CVV
5. خاص يوجّعك لـ `payment-success.html` ✅

---

## 🚀 الخطوة 5 — التحويل للدفع الحقيقي (Live)

ملي تتأكد كلشي خدّام فالتجريب:

1. فـ Telr: دير **Activation** (الزر فوق) باش تنقل الإعدادات من Test لـ Live
2. فـ Vercel: بدّل `TELR_TEST_MODE` من `1` لـ `0` → Save → Redeploy
3. سالينا! دابا الدفع حقيقي 🎉

## 🤔 أسئلة محتملة

**"العميل بغى يخلّص كاش / واتساب؟"**
كاين خيار **"Order on WhatsApp"** فنفس الصفحة — كيرسل كل تفاصيل الطلب للواتساب. خليناه لأن بعض الناس كيفضّلو هكا، وللـ pickup.

**"شنو إلا فشل الدفع؟"**
كيرجع لـ `payment-failed.html` فيها زر "جرّب مرة أخرى" + خيار الواتساب. ماكاينش أي خصم.

**"واش آمن؟"**
آه 100%. السر فـ Vercel فقط. معلومات الكارط كتجمعها Telr (ماشي موقعك) — هي PCI DSS Level 1 معتمدة.

## 📞 إلا تقطّع شي حاجة

قول ليا فأي خطوة وقفتي، وأنا نعاونك. الأخطاء الشائعة:
- نسيتي **Redeploy** بعد ما زدتي المتغيرات
- الـ Auth Key فيه مسافة زائدة (تأكد منو)
- "Allow tests from any IP" باقي **No**
